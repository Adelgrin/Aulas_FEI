# REST API

**RE**presentational

**S**tate

**T**ranfer

## Representação

* Sequência de bytes + metadados que descrevem os bytes (+

metadados que descrevem os metadados)

* Componentes REST realizam chamadas a recursos usando uma

representação que informam o estado atual ou o estado

desejado

* Metadados no formato de pares chave-valor
* Respostas podem contar tanto dados + metadados como

metadados sobre o recurso usado

## Conectores:

Funciona como diretórios, encapsulando cada recurso a enviar

Possível alterar a implementação sem o conhecimento do usuário

Stateless

### Stateless:

* O servidor não armazena estados de interações anteriores
* Cada mensagem trocada entre cliente e servidor deve conter

toda a informação necessária para realizar a requisição

* Portanto, o cliente deve manter todo estado (dados) da

aplicação
